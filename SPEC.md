# 스펙: 정책자금 컨설팅 회사 내부 전산시스템

## 완성 모습 (시나리오)

**아침 9시, 영업팀 담당자**
- 어제 소통하던 고객 5명이 "계약 완료" 상태로 변경
- 각각 클릭하면 자동으로 "계약 데이터베이스"로 이동

**10시, 관리팀 담당자**
- 대시보드 열면 "신규 업체 5건 인수인계됨" 알림 (이메일 + 시스템 내 알림)
- 각 업체의 필수 서류 체크리스트를 보며 신규 등록 프로세스 시작
- 자금 신청 진행 중인 업체들 상태를 칸반 보드(미신청 → 신청 중 → 완료)로 한눈에 파악

---

## 기능 목록

### MVP (꼭 필요)
1. **고객 데이터베이스** (영업팀)
   - 고객사명, 담당자명, 연락처, 초기상담일
   - 소통중 / 계약완료 상태
   - 담당자 할당

2. **계약 데이터베이스** (영업팀 입력 → 관리팀 조회)
   - 고객사명, 계약일, 서비스항목, 예상자금규모
   - 영업팀 쓰기, 관리팀 읽기

3. **자금신청 진행현황** (관리팀)
   - 칸반 보드 (미접수 → 서류취합중 → 신청 → 진행중 → 완료)
   - 각 건: 담당자, 마감일, 필수서류 체크리스트
   - 파일 업로드

4. **할 일 & 일정** (팀별 개인화)
   - 영업팀: 미결 계약, 소통 중 고객
   - 관리팀: 신규업체, 서류보완필요, 자금신청재실행
   - 마감일 기반 정렬

5. **자동화 플로우**
   - 영업팀 "계약완료" 클릭 → 자동으로 계약DB에서 관리팀DB로 이동
   - 관리팀에 이메일/푸시 알림

### v2+ (나중에)
- 담당자별 성과대시보드
- 마감일 3일전 자동알림
- 고객사별 타임라인
- 노트/코멘트 기능

## 1단계: Next.js 최소 동작 버전 (오늘)

**목표**: 레이아웃 + 4개 페이지가 로드되고, 더미 데이터가 표시되는 것 확인

**완료 조건**:
- [ ] `npm run dev` 실행 → localhost:3000 로드 가능
- [ ] 사이드바 메뉴 (대시보드, 고객관리, 계약관리, 진행현황)
- [ ] 각 페이지 진입 가능 + 더미 데이터 표시

**만들·수정할 파일**:
- `app/layout.tsx` — 헤더 + 사이드바 레이아웃
- `app/(dashboard)/page.tsx` — 대시보드 (간단한 통계)
- `app/(dashboard)/customers/page.tsx` — 고객 목록 (테이블)
- `app/(dashboard)/contracts/page.tsx` — 계약 목록 (테이블)
- `app/(dashboard)/funding/page.tsx` — 진행현황 (칸반 카드)
- `components/Sidebar.tsx` — 사이드바 컴포넌트
- `data/mockData.ts` — 더미 데이터

**필요한 외부 도구**: 없음 (localhost에서만 실행)

---

## 2단계: Supabase 연동

**목표**: 실제 데이터베이스에서 고객/계약/진행상황 데이터를 읽고 쓰기

**완료 조건**:
- [ ] Supabase 프로젝트 생성 + API Key 발급
- [ ] customers, contracts, funding_cases 테이블 생성
- [ ] `/api/customers`, `/api/contracts` API 라우트 작성
- [ ] 페이지에서 Supabase 데이터 렌더링 (더미 데이터 대체)

**필요한 외부 도구**: 
- **Supabase** (supabase.com 가입 → 프로젝트 생성 → API Key 복사)

---

## 3단계: 권한 관리 (Role-based Access)

**목표**: 로그인 기능 + 역할(영업팀, 관리팀)에 따라 다른 권한 부여

**필요한 외부 도구**: 
- Supabase Auth (이미 포함)
- NextAuth.js (선택: Supabase Auth 단독으로도 가능)

---

## 4단계: 자동화 트리거 (핵심!)

**목표**: 영업팀이 고객을 "계약완료"로 변경 → 자동으로 관리팀 DB로 이동 + 알림 발송

**필요한 외부 도구**: 
- Resend (resend.com 가입 → API Key 발급)

---

## 5단계: 알림 시스템

**목표**: 이메일 알림 + 시스템 내 알림 (대시보드에 표시)

---

## 6단계: 진행현황 보드 고도화

**목표**: 칸반 보드에서 드래그 & 드롭으로 상태 변경 + 서류 업로드

**필요한 외부 도구**: 
- dnd-kit (npm install로 설치)
- Supabase Storage (이미 포함)

---

## 7단계: UI/UX 다듬기

**목표**: 모바일 반응형 + 검색·필터 + 세부 화면 개선

---

## 8단계: 배포 & 운영 준비
